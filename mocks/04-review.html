3. Building meta-harnesses

✓

If the harness is the controller, the meta-harness is the controller that authors controllers. It evolves as the task and the user evolve. Two open questions follow: how do you express the search space, and how do you stay within the user's prior?

The search space is not free. Every harness atom is typed; every mutation is a delta. The Architect — an LLM stage that emits a manifest, not raw HTML — is forced to compose within a grammar. This is the difference between a model that hallucinates a UI and a model that picks atoms from a kit. The search space is not free. Every harness atom is typed; every mutation is a delta. And this is where it gets recursive. The harness watches the user write, infers a voice model, then re-asks the model to author in that voice. Imagine the loop running for a year — the doc isn't yours, it's a frozen image of how you write right now.
